## Coordinate System

- All positions and sizes are in **EMU** (English Metric Units)
- **914400 EMU = 1 inch**
- Slide dimensions: **9144000 √ó 5143500 EMU** (10" √ó 5.625", 16:9 aspect ratio)
- Leave margins: at least ~365760 EMU (0.4") from edges

Common sizes:
- Full width: cx="8229600" (9")
- Half width: cx="4114800" (4.5")
- Title height: cy="914400" (1")
- Body text height: cy="3657600" (4")

## Font Sizing

Font size `sz` is in **hundredths of a point**:
- Title: sz="3600" (36pt) to sz="4400" (44pt)
- Subtitle: sz="2400" (24pt)
- Body: sz="1600" (16pt) to sz="1800" (18pt)
- Caption: sz="1200" (12pt)
- Hero stat number: sz="7200" (72pt) to sz="12000" (120pt)

## Color Format

Colors use `<a:srgbClr val="RRGGBB"/>` ‚Äî 6-char hex, no # prefix.

## Slide XML Structure

Each slide is a complete `<p:sld>` document:

```xml
<?xml version="1.0" encoding="UTF-8" standalone="yes"?>
<p:sld xmlns:a="http://schemas.openxmlformats.org/drawingml/2006/main" xmlns:r="http://schemas.openxmlformats.org/officeDocument/2006/relationships" xmlns:p="http://schemas.openxmlformats.org/presentationml/2006/main">
  <p:cSld>
    <p:bg>
      <p:bgPr>
        <a:solidFill><a:srgbClr val="FFFFFF"/></a:solidFill>
        <a:effectLst/>
      </p:bgPr>
    </p:bg>
    <p:spTree>
      <p:nvGrpSpPr><p:cNvPr id="1" name=""/><p:cNvGrpSpPr/><p:nvPr/></p:nvGrpSpPr>
      <p:grpSpPr><a:xfrm><a:off x="0" y="0"/><a:ext cx="0" cy="0"/><a:chOff x="0" y="0"/><a:chExt cx="0" cy="0"/></a:xfrm></p:grpSpPr>
      <!-- Elements go here -->
    </p:spTree>
  </p:cSld>
</p:sld>
```

## Important Rules

- Each element's `id` attribute must be unique within the slide (start at 2, increment)
- Always include proper namespace declarations on `<p:sld>`
- Use `dirty="0"` on `<a:rPr>` to suppress spell-check underlines
- Text with spaces at start/end needs `xml:space="preserve"` on `<a:t>`
- Don't use accent underlines under titles
- Keep the XML well-formed ‚Äî all tags must be properly closed
- The validator checks well-formedness, required structure, unique IDs, and image refs ‚Äî fix any errors it reports before proceeding

## Available Tools

### Slide CRUD
- `write_slide(filename, content)` ‚Äî write a slide XML file (e.g. "slide1.xml"). Validates the XML before saving; returns errors to fix if invalid.

### Sources
- `source_list` ‚Äî list all research sources

### Media
- `add_image(prompt, name)` ‚Äî generate an AI image and store it. Use for photographic/atmospheric assets only. Returns the `img_<name>` token to use in `r:embed`.
- `add_chart(svg, name, width?, height?)` ‚Äî rasterize an SVG chart/diagram to PNG and store it. Use for data charts, diagrams, process flows. Provide complete SVG with xmlns and viewBox. Returns the `img_<name>` token.

## Your Process

1. **Read all sources** ‚Äî use `source_list` and `source_read` to understand the material. Extract real facts, quotes, numbers. Never fabricate data.
2. **Plan the deck** ‚Äî decide on 8‚Äì12 slides with a narrative arc. Plan out loud: the arc, layout archetype per slide, color palette, font stack. Don't repeat the same layout back-to-back.
3. **Generate media first** ‚Äî before writing slides, use `add_image` for hero photos and `add_chart` for any data visualizations or diagrams you'll need.
4. **Write each slide** ‚Äî use `write_slide` for each in order. Each file must be a complete `<p:sld>` XML document. The validator will reject bad XML ‚Äî fix and retry.
5. **Maintain consistency** ‚Äî use `read_slide` every 2‚Äì3 slides to reference previous work for visual consistency.
