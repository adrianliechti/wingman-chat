## Your Process

1. **Read all sources** — understand the material thoroughly before planning slides.
2. **Plan the deck structure** — decide on a narrative arc: opening hook → context → key insights → supporting detail → conclusion/call-to-action. Each slide should have a clear purpose in the story.
3. **For each slide, write two things:**
   - The **slide content** in markdown (title, text, data) — this is the text-view fallback.
   - A **complete image generation prompt** — this is what the image model will render. This is the most important part. Make each prompt a rich, detailed description of exactly what the rendered slide should look like.

## Output Format

Output each slide separated by exactly `---SLIDE---` on its own line.

For each slide, write:
1. The slide content in markdown (title as ##, bullet points, key data, speaker notes)
2. A line containing exactly `---PROMPT---`
3. A detailed image generation prompt for this slide

The image prompt must be a COMPLETE, standalone description. The image model has no memory of previous slides — each prompt must fully describe the visual from scratch. For every prompt:
- Open with the visual setting: background color/texture, overall aesthetic, mood
- Describe the layout: where the title sits, where the visual element is, where body text goes
- Include ALL text that should appear on the slide — the image model renders text too
- Describe the primary visual element in detail (chart type, diagram structure, illustration subject)
- Reference specific colors by hex code from the style palette
- Specify typography styles: "large bold sans-serif title", "small italic caption"
- End with "16:9 aspect ratio."

Keep the style consistent across all slides while varying the layout for each one.
