## Style: Deployment

Draw a **deployment / infrastructure topology** diagram — where containers run, on what infrastructure, in which environment. The audience is platform engineering, SRE, and security architecture.

Set `kind: "deployment"`.

Use these element kinds:
- `deployment-node` — every infrastructure layer: cloud provider, region, VPC, availability zone, Kubernetes cluster, namespace, ECS service, VM, server, edge POP, on-prem rack. **Nest** them via the `parent` field so the diagram reads as a tree of boxes. Tag with the technology / vendor in `technology` ("AWS eu-central-1", "Kubernetes 1.30", "EKS", "Cloudflare", "VMware vSphere", "Azure App Service").
- `container` — the running application / database / queue instances **deployed onto** a deployment node. Their `parent` is the leaf deployment node. Tag `technology` with the runtime stack ("Java 21 / Spring Boot", "PostgreSQL 15 (RDS)", "Redis 7").
- `external-system` — managed services or external endpoints the deployment depends on (managed databases, SaaS, CDN, identity providers). Place outside the deployment tree.

Use `group` of `kind: "deployment-group"` for logical zones that aren't strictly nested (e.g. "Production environment", "DR site", "DMZ").

Design discipline:
- Default depth: **Cloud → Region → Cluster/VPC → Compute unit → Container**. Don't go deeper unless the source demands it.
- **Show DR / multi-region only if the source mentions it.** Inventing a DR site is a hard `inferred: true` — DR posture is a business decision, not a default.
- For each deployment node specify the security boundary it represents in the description: "public subnet", "private subnet", "PCI-DSS scope", "data residency: CH only".
- Tag the runtime: container image registry, autoscaling policy, instance type if the source supports it.
- Relations carry the protocol: "Ingress traffic (HTTPS/443)", "VPC peering", "PrivateLink", "Cross-region replication (async)".

What to mark `inferred: true`:
- Any node the source does not explicitly name (e.g. you assumed AWS when the source said "the cloud").
- Reasonable security defaults (private subnets, WAF, NAT gateway) that weren't called out.

Do **not** use `person`, `system`, `actor`, `entity`, or `component` in this style.
