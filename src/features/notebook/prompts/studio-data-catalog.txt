## Design Discipline (applies to every style)

1. **Pick a credible target domain model.** From the sources, decide which datasets a senior data architect would create. If the source mentions "we ingest trades and produce risk reports", design the realistic set: `trade_capture.trade_events` (Kafka) → `dwh.fact_trades` (Snowflake, daily) → `risk.exposure_eod` (Snowflake, daily) → `reporting.fr_y14_extract` (S3, monthly).
2. **Owners are first-class.** Every dataset needs an `owner` (business — "Trade Operations Lead", "Credit Risk Head") AND a `steward` (technical — "Trading Platform Engineering", "DataOps"). If the source doesn't name them, infer the most plausible function and tag `inferred: true`.
3. **Classify everything.** Every dataset gets `sensitivity` and `regulatoryTags`. PII-bearing datasets get `GDPR-PII`. Trade data gets `MiFID-trade-data`. Credit risk inputs get `BCBS-239-CDE`. Payment card data gets `PCI-scope`. Capital adequacy inputs get `CRR-CCAR`. When in doubt, classify higher rather than lower.
4. **Lineage from physical systems.** Lineage `LineageNode` of `kind: "job"` should be a named transformation with `technology` set (dbt, Airflow, Spark, Fivetran, Talend, Informatica). Avoid generic "ETL Job" — pick a tool that fits the stack implied by the sources.
5. **Glossary terms map to datasets.** Every glossary term should have at least one dataset realising it (`datasets: [...]`). Terms that don't get realised somewhere are theatre.
6. **Contracts are commitments, not aspirations.** Each contract's `qualityRules` should be testable ("no NULL `customer_id`", "checksum vs. SoR within 0.1%"), and `terms` should be measurable ("Available 24×7 except Sun 02:00-04:00 UTC", "Freshness: T+1 by 06:00 UTC").
7. **Compactness.** Aim for **8–20 datasets**, **6–15 glossary terms**, **lineage to cover the in-scope flow**, **contracts for the 3–8 most critical datasets**. More is not better.
8. **Stable, descriptive ids.** Datasets `customer_master`, `trade_events`, `exposure_eod`. Terms `term_counterparty`, `term_trade_date`. Lineage nodes `job_trade_ingest`, `dataset_trade_events`. Edges `edge_trade_ingest`.
